# 安全政策

## 支持范围

安全修复只面向 GitHub Releases 中最新发布版本。使用旧版本时，请先按 [INSTALL.md](INSTALL.md) 的手动更新流程升级到最新版，再验证问题是否仍存在。

## 报告安全问题

如果仓库启用了 GitHub Private Vulnerability Reporting，请优先使用私密漏洞报告。若该入口不可用，请先提交一个不含技术细节、凭据或个人数据的公开 issue，请维护者提供私密沟通方式。不要在公开 issue、讨论、提交、截图或日志摘录中披露漏洞利用细节、凭据或个人数据。

公开安装问题只应包含：固定错误码、失败阶段、组件版本、四项健康检查结果，以及人工检查并脱敏后的少量 `data\logs\install.log` 摘录。

## 禁止公开上传

以下内容不得公开上传或粘贴：

- 整个 `data` 目录或其压缩包；
- 原始日志，包括完整 `data\logs\install.log`、`launcher.log` 和 `bridge.log`；
- `FIRST_LOGIN.txt`；
- WeFlow 配置、AstrBot 配置或桥接的真实配置；
- 模型 API Key、访问令牌、认证令牌、密码或其他凭据；
- 微信或聊天数据库、消息内容、联系人信息；
- 图片、文件等聊天附件；
- 包含凭据、个人数据、聊天内容或私人路径的截图。

原始日志应视为私密资料。只有经过人工逐行检查、删除用户名、路径、令牌、联系人与消息内容后的短摘录才可以分享。

`data\logs\bridge.log` 有意记录私聊联系人、群名与群成员、收到的完整正文，以及 Bot 尝试发送的完整正文和发送状态。Web 控制面板会在本机回环地址显示这些结构化聊天记录，但不会返回其他原始运行日志。日志和面板都只能在受信任的本机账户下使用；不要通过公开 issue、网盘或整包诊断附件分享，并应根据实际需要定期删除旧日志。

如果任何凭据曾出现在 issue、提交、截图或日志摘录中，不要只删除内容：应立即在对应服务中撤销或轮换该凭据，然后清理公开记录。

本政策适用于 AkashaBot WeFlow Bridge 的 MIT 许可代码。WeFlow 和 AstrBot 的上游安全问题请同时遵循各自项目的报告政策；本项目不代表其官方。
